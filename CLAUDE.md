# CLAUDE.md — Bark

## What is Bark?

Minimal NIP-07 browser extension that bridges `window.nostr` to a remote NIP-46
signer. Works with any NIP-46 bunker; persona features require Heartwood.

## Build and test

```bash
npm install
npm run build        # esbuild → dist/ (chromium); build:firefox, build:safari, build:all
npm test             # vitest unit tests
npm run e2e:chromium # Playwright e2e (builds first; includes deterministic local NIP-46 signer)
npm run package:all  # build + zip all three browser targets
```

## Architecture

Source files in `src/`:

| File | Role |
|------|------|
| `provider.js` | MAIN-world content script (Chromium/Firefox; Safari falls back to script-tag injection). Implements `window.nostr` API. Posts messages to content script. |
| `content-script.js` | Bridge. Relays postMessage ↔ chrome.runtime.sendMessage. Validates origin and structure. Retries through MV3 worker wake-up. |
| `background.js` | Service worker. NIP-46 connection via nostr-tools BunkerSigner, nostrconnect pairing, approval queue, policy checks, keep-alive pings, error sanitisation, connection state. |
| `policy.js` | Pure policy engine: allow/ask/deny per method, kind, and site. |
| `popup.js` | Popup UI. Instance management, QR pairing, policy settings, relay display, reconnection. |
| `popup.html` | Popup markup and styles. Dark theme, 320px wide. |
| `approve.js` / `approve.html` | Approval window for ask-policy requests (allow once / trust site / deny). |
| `diagnostic.html` | Extension-internal page used by e2e for storage seeding and runtime messaging. |

Per-target manifests are generated by `esbuild.config.js` from `src/manifest.json`.

### Message flow

```
Web page → provider.js (postMessage) → content-script.js (chrome.runtime.sendMessage)
→ background.js (BunkerSigner → NIP-46 relay → Heartwood) → response back
```

### Message types (chrome.runtime)

- `bark-request` — NIP-07/NIP-44/Heartwood RPC call
- `bark-reset` — tear down connection (disconnect or URI change)
- `bark-status` — query connection state, relay health, Heartwood mode, auth_url
- `bark-pair` / `bark-switch` / `bark-remove` — instance management
- `bark-nostrconnect-start` / `-status` / `-cancel` — QR pairing flow
- `bark-approval-query` / `bark-approval-response` — approval window RPC
- `bark-prime-signer` — signer health check (signs a kind 22242 probe)

## Conventions

- **British English** — colour, initialise, behaviour
- **No local key material** — the extension never touches private keys
- **Pure functions for testable logic** — parseMethod, validators, sanitiseError, buildHeartwoodArgs are all exported and unit tested
- **Commit messages** — `type: description` (feat:, fix:, test:, docs:)

## Key dependency

`nostr-tools` — provides `BunkerSigner`, `parseBunkerInput`, `SimplePool`, crypto utilities. Pinned at `^2.23.0`.
